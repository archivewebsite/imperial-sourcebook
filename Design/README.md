# Design

`Design` stores design study material, interface references, visual inspiration, and reusable observations about how products look and feel.

## Current Buckets

- `Design Class`: design lessons, principles, and study material.
- `Design Inspiration`: UI references, palettes, screenshots, and examples worth borrowing from later.

## What Belongs Here

- design theory and class notes
- palette examples and visual reference notes
- product screenshots or UI observations with a clear reason for saving them
- reusable design lessons that should outlive a single project

## What Does Not

- active design project planning that belongs in `D:\Imperial Records\Projects`
- unstable design experiments that belong in `D:\Imperial Records\Labs`
- broad non-design references that belong in `Notes`
- raw image files without markdown context

## How To Use It

- save the specific design quality worth remembering, not just the source
- keep examples close to the pattern they demonstrate
- move active work back to Records and keep only the reusable lesson here
